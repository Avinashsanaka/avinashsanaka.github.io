Featured Case Study

Engineering a repeatable data migration platform for high-trust case data.

At Kunuwanimano Child & Family Services, I built a Python-based migration workspace around legacy Ingres data, CSV/workbook exports, and strict downstream template requirements. The work turned undocumented joins, merged person IDs, narrative text fields, and parent-child dependencies into documented, repeatable migration pipelines.

Scope Organized migration logic across Person, Intake, Ongoing, and Resource domains, with script-to-table mapping, ER diagrams, run logs, and status tracking for conversion and validation work.
Design Created deterministic KUNU-* unique IDs, parentUniqueID rules, stable sequence patterns, CSV output standards, date formatting, worker-name normalization, and duplicate-handling conventions.
Safeguards Resolved moved/merged person IDs through ind_moved_info, checked duplicate uniqueID values, validated missing parent links, and preserved narrative text by fetching textnum rows in line order.
Impact Caught 4,027 orphaned records before production, found roughly 20 categories of bad data, and fixed a silent multi-line narrative truncation issue before it could corrupt migrated records.
